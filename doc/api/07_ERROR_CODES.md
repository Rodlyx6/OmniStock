# 错误码文档

## 错误码规范

系统采用统一的错误码规范，便于前端处理和用户提示。

### 错误码格式

```
{
  "code": 错误码,
  "message": "错误信息",
  "data": null,
  "timestamp": "2026-03-21T10:30:00Z"
}
```

---

## 错误处理示例

### 前端处理

```javascript
// 库存不足错误
if (response.code === 5001) {
  message.error(`库存不足，可用库存: ${response.data.availableQuantity}`);
}

// 版本冲突错误（自动重试）
if (response.code === 5002) {
  // 自动重试，最多3次
  retryCount++;
  if (retryCount < 3) {
    // 重新调用接口
  } else {
    message.error('操作失败，请稍后重试');
  }
}

// Token过期错误
if (response.code === 2003) {
  // 清除Token，跳转到登录页
  localStorage.removeItem('token');
  router.push('/login');
}

// 权限不足错误
if (response.code === 2006) {
  message.error('您没有权限执行此操作');
}
```

### 后端处理

```java
// 库存不足异常
if (availableQuantity < requestQuantity) {
  throw new BusinessException(5001, "库存不足", 
    new InventoryErrorData(availableQuantity, requestQuantity));
}

// 版本冲突异常
if (updateCount == 0) {
  throw new BusinessException(5002, "库存版本冲突，请重试");
}

// 资源不存在异常
if (sku == null) {
  throw new BusinessException(3002, "SKU不存在");
}

// 参数验证异常
if (quantity <= 0) {
  throw new BusinessException(1001, "参数验证失败: 数量必须大于0");
}
```

## 常见错误场景

### 场景1：库存扣减失败

**错误码：** 5001

**原因：** 库存不足

**解决方案：**
1. 检查库存是否充足
2. 查询库存流水，确认库存变化
3. 如果库存确实不足，提示用户等待补货

### 场景2：版本冲突

**错误码：** 5002

**原因：** 并发扣减导致版本冲突

**解决方案：**
1. 自动重试（最多3次）
2. 如果重试仍失败，返回库存不足错误
3. 监控版本冲突频率，如果过高，考虑优化并发策略

### 场景3：Token过期

**错误码：** 2003

**原因：** Token已过期

**解决方案：**
1. 清除本地Token
2. 跳转到登录页
3. 用户重新登录

### 场景4：权限不足

**错误码：** 2006

**原因：** 用户没有访问权限

**解决方案：**
1. 检查用户角色和权限
2. 提示用户联系管理员申请权限
3. 隐藏无权限的功能按钮

### 4. 前端重试策略

```javascript
// 指数退避重试
const retryWithBackoff = async (fn, maxRetries = 3) => {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (error.code === 5002 && i < maxRetries - 1) {
        // 版本冲突，重试
        await sleep(Math.pow(2, i) * 100); // 100ms, 200ms, 400ms
      } else {
        throw error;
      }
    }
  }
};
```
